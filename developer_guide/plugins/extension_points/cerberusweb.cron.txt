cerberusweb.cron
----------------

The **cerberusweb.cron** extension point allows you to create new :doc:`Scheduled Tasks`.

Plugin Manifest
===============

In :doc:`plugin.xml </developer_guide/plugins/plugin.xml>`:

.. code-block:: xml
    
    <!-- Cron -->
    <extension point="cerberusweb.cron">
        <id>example.cron</id>
        <name>Example Maintenance</name>
        <class>
            <file>api/App.php</file>
            <name>ExampleCron</name>
        </class>
        <params/>
    </extension>

Implementation
==============
.. code-block:: php

    <?php
    class ExampleCron
        extends CerberusCronPageExtension {

            public function run() {}
            
            public function configure($instance) {}
            
            public function saveConfigurationAction() {}
    };

**run()**
    This method is called when the platform determines it is time for your scheduled task to run.
    
**configure($instance)**
    Displays the configuration area for this scheduled task on the setup -> Scheduler page. Your implementation is passed an ``$instance`` parameter with all the information about the scheduled task. See the documentation on :php:class:`CerberusCronPageExtension` for more information on the data and methods available to each scheduled task.
    
**saveConfigurationAction()**
    Saves the config when an admin clicks ``save`` in the configuration area for this scheduled task.

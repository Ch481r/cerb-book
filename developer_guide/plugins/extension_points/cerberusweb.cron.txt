cerberusweb.cron
------------------------

The **cerberusweb.cron** extension point allows you to create new :doc:`Scheduled Tasks`.

Plugin Manifest
===============

In :doc:`plugin.xml </developer_guide/plugins/plugin.xml>`:

.. code-block:: xml
    
    <!-- Cron -->
    <extension point="cerberusweb.cron">
        <id>example.cron</id>
        <name>Example Maintenance</name>
        <class>
            <file>api/App.php</file>
            <name>ExampleCron</name>
        </class>
        <params/>
    </extension>

Implementation
==============
.. code-block:: php

    <?php
    abstract class CerberusCronPageExtension
        extends DevblocksExtension {    
            /**
             * runs scheduled task
             *
             */
            function run() {}
            
            public function configure($instance) {}
            
            public function saveConfigurationAction() {}
    };

**run()**
    This method is called when the platform determines it is time for the scheduled task to run.
    
**configure($instance)**
    Displays the configuration area for this scheduled task on the setup -> Scheduler page. The method is passed an ``$instance`` parameter with all the information about the scheduled task. See the documentation on :doc:`CerberusCronPageExtension </developer_guide/devblocks/cerberuscronpageextension>` for more information on the data and methods available to each scheduled task.
    
**saveConfigurationAction()**
    Saves the config when an admin clicks ``save`` in the configuration area for this scheduled task.

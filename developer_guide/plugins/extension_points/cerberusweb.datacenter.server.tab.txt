cerberusweb.datacenter.server.tab
---------------------------------

The **cerberusweb.datacenter.server.tab** extension point allows you to create new tabs on the ``Servers`` page.

Plugin Manifest
===============

In :doc:`plugin.xml </developer_guide/plugins/plugin.xml>`:

.. code-block:: xml
    
    <!-- Activity Tab -->
    <extension point="cerberusweb.datacenter.server.tab">
        <id>example.datacenter.server.tab</id>
        <name>Example Datacenter Server Tab</name>
        <class>
            <file>api/App.php</file>
            <name>ExampleDatacenterServerTab</name>
        </class>
        <params>
            <param key="uri" value="example" />
            <param key="title" value="example.datacenter.server.tab" />
        </params>
    </extension>

**<param key="uri" value="example">**
    Binds the tab to the ``example`` url.

**<param key="title" value="example.title">**
    Sets the text of the tab. Devblocks will automatically pass this string through the :doc:`translation service` before rendering it, so you should use a namespaced translation string whenever possible. You can use literal text if you wish, though we advise against it.

Implementation
==============
.. code-block:: php

    <?php
    class ExampleDatacenterServerTab
        extends Extension_ServerTab {
            
            function showTab(Model_Server $server) {}
            
            function saveTab() {}
    };

**showTab()**
    This method is called when a worker visits your tab on Server object's page. The server currently being displayed can be accessed via the ``$server`` variable. See the documentation on :php:class:`Model_Server` for more information on the data and methods available for each server.

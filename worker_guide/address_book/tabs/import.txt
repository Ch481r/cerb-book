Import
------

The **Import** tab is where you import :doc:`Organizations</worker_guide/address_book/organizations/index>` and :doc:`Addresses</worker_guide/address_book/orges/index>` from a CSV file.

.. figure:: /images/worker_guide/address_book/tabs/import_tab.png
    :alt: The import tab.
    :figclass: bordered

You can specify whether you want to import :guilabel:`Organizations` or :guilabel:`Contacts`, and the location of the CSV file to import.

.. figure:: /images/worker_guide/address_book/tabs/import/import.png
    :alt: The import section.
    :figclass: bordered

Clicking the upload button will upload the CSV and allow you to select what data to insert into the database.

.. figure:: /images/worker_guide/address_book/tabs/import/upload_button.png
    :alt: The upload button.
    :figclass: bordered

Organizations
=============

Selecting the :guilabel:`Organizations` option allows you to import Organizations from a CSV file.

.. figure:: /images/worker_guide/address_book/import/organizations.png
    :alt: The organizations screen.
    :figclass: bordered

Columns and Fields
~~~~~~~~~~~~~~~~~~

The first column contains the columns present in the CSV file you uploaded.

.. figure:: /images/worker_guide/address_book/import/org_columns.png
    :alt: The organization columns.
    :figclass: bordered

The second column allows you to specify which field each column in your CSV should be saved to.

.. figure:: /images/worker_guide/address_book/import/org_fields.png
    :alt: The organization fields.
    :figclass: bordered

Options
~~~~~~~

There are two options that can be toggled when importing Organizations.

.. figure:: /images/worker_guide/address_book/import/org_options.png
    :alt: The organization options.
    :figclass: bordered

Import the first row
""""""""""""""""""""

Toggling the :guilabel:`Import the first row` option allows you to import the first row of data from the CSV file.

.. figure:: /images/worker_guide/address_book/import/org_options_import.png
    :alt: The import first row option.
    :figclass: bordered

Blank values for custom fields should clear the field
"""""""""""""""""""""""""""""""""""""""""""""""""""""

Toggling the :guilabel:`Blank values for custom fields should clear the field` option allows you to clear Custom Fields for existing records when the import is set to :guilabel:`Check for duplicates` and the field for the row in the CSV is blank. 

.. figure:: /images/worker_guide/address_book/import/org_options_blank.png
    :alt: The clear custom fields option.
    :figclass: bordered

Check for duplicates
~~~~~~~~~~~~~~~~~~~~

The :guilabel:`Check for duplicates using` option allows you to specify which field should be used for checking duplicates.

.. figure:: /images/worker_guide/address_book/import/org_check_dupes.png
    :alt: The check dupes setting.
    :figclass: bordered

Selecting the :guilabel:`-- don't check for duplicates --` option from the select menu will force all rows present in the CSV to be inserted into the database.
